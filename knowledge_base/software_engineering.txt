ROLE: Software Engineer / Software Developer
DOMAIN: Technology, IT, Product Development

=== COMMON INTERVIEW TOPICS ===
1. Data Structures & Algorithms
   - Arrays, Linked Lists, Trees, Graphs, Hash Tables, Heaps
   - Sorting: QuickSort, MergeSort, HeapSort
   - Searching: Binary Search, BFS, DFS
   - Dynamic Programming, Greedy Algorithms, Backtracking

2. System Design
   - Scalability, Load Balancing, Caching (Redis, Memcached)
   - Microservices vs Monolith architecture
   - API Design (REST, GraphQL, gRPC)
   - Database design: SQL vs NoSQL, Sharding, Indexing
   - Message Queues (Kafka, RabbitMQ)
   - CAP Theorem, eventual consistency

3. Object-Oriented Programming
   - SOLID principles
   - Design Patterns: Singleton, Factory, Observer, Strategy, Decorator
   - Inheritance vs Composition

4. Languages & Frameworks
   - Python: Flask, Django, FastAPI
   - JavaScript/TypeScript: React, Node.js, Next.js
   - Java: Spring Boot
   - Cloud: AWS, GCP, Azure, IBM Cloud

=== SAMPLE INTERVIEW QUESTIONS ===
TECHNICAL:
Q: Explain the difference between a stack and a queue. When would you use each?
Q: How would you design a URL shortening service like bit.ly?
Q: What is the time complexity of binary search? Explain why.
Q: Describe the difference between SQL JOIN types.
Q: How do you handle race conditions in concurrent programming?
Q: What is a RESTful API? What are its constraints?
Q: Explain dependency injection and its benefits.
Q: How would you optimize a slow database query?
Q: What is Docker and why is containerization useful?
Q: Explain the CAP theorem with an example.

BEHAVIORAL:
Q: Tell me about a time you debugged a difficult production issue.
Q: Describe a project where you had to learn a new technology quickly.
Q: How do you handle disagreements with teammates about technical decisions?
Q: Tell me about your most challenging coding project.

=== MODEL ANSWERS (STAR FORMAT) ===
Situation → Task → Action → Result

For "Tell me about a time you debugged a difficult issue":
- Situation: Production API was returning 500 errors for 10% of users
- Task: Identify root cause within 2 hours during peak traffic
- Action: Used distributed tracing (Jaeger), narrowed to a race condition in cache invalidation, implemented a mutex lock, deployed hotfix
- Result: Error rate dropped to 0% within 15 minutes, wrote post-mortem to prevent recurrence

=== PREPARATION TIPS ===
1. Practice LeetCode Medium difficulty problems daily (30–45 min)
2. Study system design: "Designing Data-Intensive Applications" (Kleppmann)
3. Review your past projects — prepare 3–5 STAR stories
4. Understand Big-O notation and be able to analyze your own code
5. Practice explaining code aloud — interviewers value communication
6. Research the company's tech stack before the interview
7. Prepare questions to ask the interviewer (team culture, deployment cadence, tech debt)
8. For system design: always clarify requirements, estimate scale, then design

=== SKILLS TO HIGHLIGHT ===
- Problem decomposition and clean code practices
- Experience with version control (Git flow, PRs, code review)
- CI/CD pipelines and DevOps awareness
- Testing: unit, integration, end-to-end
- Debugging tools and profiling
- Collaboration tools: Jira, Confluence, Slack
