ROLE: Data Scientist / ML Engineer / AI Engineer
DOMAIN: Data, Machine Learning, Artificial Intelligence, Analytics

=== COMMON INTERVIEW TOPICS ===
1. Statistics & Probability
   - Hypothesis testing, p-values, confidence intervals
   - Bayes' theorem, conditional probability
   - Distributions: Normal, Binomial, Poisson
   - A/B testing design and analysis
   - Central limit theorem

2. Machine Learning

   - Unsupervised: K-Means, DBSCAN, PCA, t-SNE
   - Deep Learning: CNNs, RNNs, Transformers, Attention mechanism
   - Evaluation: Accuracy, Precision, Recall, F1, ROC-AUC, RMSE
   - Overfitting/underfitting, regularization (L1/L2, Dropout)
   - Cross-validation, hyperparameter tuning

3. Data Engineering & Processing
   - Pandas, NumPy, SQL for data manipulation
   - Feature engineering, feature selection
   - Handling missing data, outlier detection
   - ETL pipelines, data warehousing concepts

4. MLOps & Deployment
   - Model versioning (MLflow, DVC)
   - Model serving (Flask, FastAPI, TorchServe)
   - Monitoring for data drift and model decay
   - IBM Watson Studio, IBM watsonx.ai

5. NLP & LLMs
   - Tokenization, embeddings, TF-IDF, Word2Vec, BERT
   - Fine-tuning vs prompt engineering
   - RAG (Retrieval-Augmented Generation)
   - IBM Granite models for enterprise AI

=== MODEL ANSWERS ===
For "How do you handle an imbalanced dataset?":
- Collect more data from the minority class if possible
- Resampling: oversample minority (SMOTE) or undersample majority
- Use class-weight parameters in sklearn models
- Choose appropriate metrics: F1, ROC-AUC instead of raw accuracy
- Try anomaly detection approaches for extreme imbalance

=== PREPARATION TIPS ===
1. Master pandas and SQL — most take-home tasks involve data wrangling
2. Be able to implement common algorithms from scratch (logistic regression, k-means)
3. Know your metrics cold — when to use what, and why
4. Prepare 2–3 end-to-end project walkthroughs (problem → data → model → deploy)
5. Study statistics fundamentals — many interviews test this heavily
6. For NLP roles: understand embeddings, fine-tuning, and prompt engineering
7. Research the company's data products and what scale they operate at
8. Practice case studies: frame the problem, identify data needs, propose model, discuss tradeoffs
